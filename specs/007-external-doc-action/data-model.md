# Data Model: External Documentation Action

## Entities

### ActionInputs

The configuration interface between the caller workflow and the composite action.

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| `github-token` | string (secret) | yes | — | GitHub token for API access, PR comments, git push |
| `aicore-service-key` | string (secret) | yes | — | SAP AI Core service key for LLM calls |
| `aicore-resource-group` | string (secret) | yes | — | SAP AI Core resource group |
| `pr-number` | number | yes | — | Pull request number to process |
| `pr-branch` | string | yes | — | Head branch of the pull request |
| `base-branch` | string | no | `master` | Base branch to diff against |
| `scope-mapping` | JSON string | no | `{}` | File glob → doc file mapping (see ScopeMapping) |
| `always-update` | string | no | `"CHANGELOG.md"` | Space-separated list of files always regenerated |
| `templates-dir` | string | no | `""` | Path to project-specific templates (overrides defaults) |
| `docs-dir` | string | no | `"docs"` | Output directory for generated documentation |
| `agent-command` | string | no | `"npx alexi agent"` | CLI command to invoke the LLM agent |
| `agent-max-iterations` | number | no | `30` | Max agent tool-call iterations |
| `agent-effort` | string | no | `"high"` | Agent effort level: `low`, `medium`, `high` |
| `agent-tools` | string | no | `"read,write,edit,glob,grep"` | Comma-separated list of agent tools |
| `commit-tag` | string | no | `"[doc-bot]"` | Tag appended to commit messages for bot detection |
| `commit-message-prefix` | string | no | `"docs: auto-generate documentation"` | Prefix for commit messages |
| `force-regeneration` | boolean | no | `false` | Force full regeneration regardless of changes |
| `system-prompt` | string | no | *bundled default* | System prompt for the LLM agent |

### ActionOutputs

Values propagated back to the caller workflow.

| Field | Type | Description |
|-------|------|-------------|
| `docs-updated` | string | Space-separated list of doc files that were updated |
| `skipped` | boolean string | `"true"` if no code changes and not forced |
| `success` | boolean string | `"true"` if generation completed without errors |
| `commit-sha` | string | SHA of the documentation commit (empty if no changes) |
| `files-changed` | number string | Count of documentation files modified |

### ScopeMapping

JSON structure mapping file change patterns to documentation targets. Parsed from the `scope-mapping` input.

```json
{
  "<glob-pattern>": ["<doc-file-path>", ...],
  ...
}
```

**Example (Alexi project)**:

```json
{
  "src/cli/**": ["docs/ARCHITECTURE.md", "docs/API.md"],
  "src/core/**": ["docs/ARCHITECTURE.md", "docs/API.md"],
  "src/core/router*": ["docs/ROUTING.md"],
  "src/config/routing*": ["docs/ROUTING.md"],
  "src/providers/**": ["docs/PROVIDERS.md"],
  "package.json": ["docs/CONFIGURATION.md"],
  "tsconfig.json": ["docs/CONFIGURATION.md"],
  "src/config/**": ["docs/CONFIGURATION.md"],
  "tests/**": ["docs/TESTING.md"],
  ".github/workflows/**": ["docs/AUTOMATION.md"],
  "scripts/**": ["docs/AUTOMATION.md"]
}
```

**Resolution rules**:
1. For each changed file from `git diff`, test against all glob patterns
2. Collect the union of all matched doc files
3. Deduplicate the result
4. Append `always-update` files
5. The resulting set is `DOCS_TO_UPDATE`

### TemplateSet

Ordered collection of markdown templates assembled into the LLM prompt.

| Order | File | Type | Description |
|-------|------|------|-------------|
| 1 | `01-header.md` | Static | Project context, structure, tech stack |
| 2 | `02-changed-files-header.md` | Static | Section header for changed files |
| 3 | *analysis.md* | Dynamic | Generated by analysis step |
| 4 | `03-commits-header.md` | Static | Section header for commits |
| 5 | *commits.md* | Dynamic | Generated by commit log step |
| 6 | `04-diff-header.md` | Static | Section header for code diff |
| 7 | *diff.md* | Dynamic | Generated by diff step |
| 8 | `05-scope-header.md` | Static | Section header for documentation scope |
| 9 | *scope.md* | Dynamic | Generated by scope determination |
| 10 | `06-requirements.md` | Static | General documentation requirements |
| 11 | `07-conditional/*.md` | Conditional | Included based on scope matches |
| 12 | `08-footer.md` | Static | Quality requirements, output format |

**Template resolution order** (per file):
1. `${templates-dir}/<filename>` (project override) — if `templates-dir` is set and file exists
2. `${github.action_path}/templates/<filename>` (action bundled default)

### DocumentationScope

Computed at runtime. Determines which doc files to regenerate.

| Field | Type | Description |
|-------|------|-------------|
| `docs_to_update` | string[] | List of file paths to regenerate |
| `scope_flags` | Record<string, boolean> | Map of scope categories to their activation status |
| `conditional_templates` | string[] | List of 07-conditional/ templates to include |
| `has_code_changes` | boolean | Whether any non-doc files changed |

### GenerationRun

Represents a single execution of the action.

| Field | Type | Description |
|-------|------|-------------|
| `pr_number` | number | Pull request being processed |
| `pr_branch` | string | Head branch |
| `base_branch` | string | Base branch for diffing |
| `last_doc_commit` | string | SHA of the most recent bot commit (for incremental analysis) |
| `changed_files` | string[] | Files changed since last doc commit |
| `docs_to_update` | string[] | Computed documentation scope |
| `agent_exit_code` | number | LLM agent process exit code |
| `validation_warnings` | string[] | Markdown lint warnings |
| `committed` | boolean | Whether a commit was created |
| `pushed` | boolean | Whether changes were pushed |

## Relationships

```
ActionInputs ──────► ScopeMapping (parsed from JSON input)
ActionInputs ──────► TemplateSet (templates-dir overrides defaults)
ScopeMapping + git diff ──────► DocumentationScope
DocumentationScope ──────► TemplateSet (selects conditional templates)
TemplateSet ──────► GenerationRun (assembled prompt drives agent)
GenerationRun ──────► ActionOutputs (results propagated to caller)
```

## State Transitions

The action progresses through a linear pipeline:

```
[Validate Inputs]
    │
    ▼
[Analyze Changes] ──── has_code_changes=false & !force ────► [Skip + Comment] ──► EXIT
    │
    ▼ (has changes or forced)
[Determine Scope]
    │
    ▼
[Collect Commits + Diff]
    │
    ▼
[Assemble Prompt] (templates + dynamic content)
    │
    ▼
[Invoke Agent] ──── failure (transient) ────► [Retry] (up to 2 attempts)
    │                                              │
    │ success                                      │ permanent failure
    ▼                                              ▼
[Validate Markdown]                          [Post Failure Comment] ──► EXIT
    │
    ▼
[Commit + Push]
    │
    ▼
[Post Success Comment]
    │
    ▼
[Upload Artifacts] ──► EXIT
```
