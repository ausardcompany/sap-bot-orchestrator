git add docs/ROUTING.md docs/PROVIDERS.md docs/CONFIGURATION.md

## Verification Checklist

After the workflow runs on the test PR:

- [ ] **Trigger**: Workflow triggers on PR open
- [ ] **Prompt**: Check `kilo-prompt.md` artifact — project structure matches actual `src/`, all 16 CLI commands listed, no duplicate Mermaid instructions
- [ ] **Agent**: Bot output log shows successful tool calls (read + write) for scoped doc files
- [ ] **File casing**: `ls -la docs/` on CI shows `ARCHITECTURE.md` (not `architecture.md`)
- [ ] **New files**: `ROUTING.md`, `PROVIDERS.md`, `CONFIGURATION.md` exist and have content (not just seed text)
- [ ] **No unmanaged staging**: `permission-system.md` etc. are NOT in the commit diff
- [ ] **Commit message**: Commit uses `[alexi-bot]` tag
- [ ] **Push**: Changes are pushed to the PR branch
- [ ] **PR comment**: Success comment appears with scope summary
- [ ] **Artifacts**: `kilo-prompt.md`, `bot-output.log`, `analysis.md`, `scope.md`, `diff.md`, `commits.md` all uploaded
- [ ] **Validation**: PR comment includes validation results (or "all clear")
- [ ] **No regressions**: `npm test` still passes on the branch

## Manual Workflow Dispatch Test

After the automatic test, verify force regeneration:

1. Go to Actions → Documentation Update → Run workflow
2. Set PR number to the test PR
3. Set `force_full_regeneration: true`
4. Verify ALL doc files are regenerated (not just those matching changed modules)

## Rollback

If the redesign breaks the workflow:
1. Revert the workflow commit: `git revert <commit>`
2. The templates and seed files are harmless and can remain
3. The case rename (`ARCHITECTURE.md`) should remain (it's the correct convention)
